# no_sql_AP_N3_C1
Bases de datos NO estructuradas, proyecto unidad 3

<h1>Proceso de Creación de Backend + MongoDB <img src='https://www.google.com/imgres?q=mongodb%20icono&imgurl=https%3A%2F%2Fimages.icon-icons.com%2F2415%2FPNG%2F512%2Fmongodb_original_wordmark_logo_icon_146425.png&imgrefurl=https%3A%2F%2Ficon-icons.com%2Fes%2Ficono%2Fmongodb-original-marca-logo%2F146425&docid=22SKF9O0eaX6YM&tbnid=jpOCvjrf9tvBKM&vet=12ahUKEwjNlfXPpO6UAxXBLrkGHWhMBBEQnPAOegQIGxAB..i&w=512&h=512&hcb=2&ved=2ahUKEwjNlfXPpO6UAxXBLrkGHWhMBBEQnPAOegQIGxAB' style='width:50px;height:50px;'></h1>

<ol>
    <li>Instalar la última versión de Node.JS</li>
    <li><ul><li><a src='https://nodejs.org/dist/v26.3.0/node-v26.3.0-x64.msi'>Link de descarga</a></li></ul></li>
    <li>Mediante el terminal, nos ubicamos en la carpeta donde se creará el servidor backend.</li>
    <li>Ejecutamos el comando: <code>npm init -y</code></li>
    <li>Posteriormente, ejecutamos el comando: <code>npm install express cors mongoose body-parser</code></li>
    <li>Si no se reconocen los comandos NPM luego de instalar NODE, probablemente VSCODE estaba abierto mientras se instaló NODE y debemos reiniciar VSCODE para que reconozca los cambios.</li>
    <li>Si el error es porque la ejecución de scripts está deshabilitada en el sistema, ejecutaremos el siguiente comando en el terminal: <code>Set-ExecutionPolicy -ExecutionPolicy RemoteSigned -Scope CurrentUser</code></li>
    <li>Una vez ejecutado el script, deberíamos poder ejecutar los scripts NPM anteriores.</li>
</ol>
